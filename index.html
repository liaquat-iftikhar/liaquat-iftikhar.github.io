Scalable Session Revenue Data Pipeline

Built a scalable ETL pipeline for session-level revenue (ads + IAP), enabling accurate LTV/ROAS. Eliminated full table scans via smart incremental processing — reducing compute costs significantly.

↓Compute Costs
RTRevenue Metrics
LTVROAS Enabled
View full case study ↓
Problem

Revenue data was fragmented, late-arriving, and computationally expensive to process. Existing pipelines relied on full table scans — causing high costs, slow execution, and unreliable LTV and ROAS metrics across millions of gaming sessions.

Solution
  • Smart Incremental Processing (Delta Lake CDF) — built a dynamic mechanism to detect late-arriving data and recompute only affected partitions, eliminating unnecessary reprocessing
  • Advanced PySpark Optimisation — resolved heavy range-overlap bottlenecks (sessions crossing midnight) using partition pruning, generated columns, and Z-ordering
  • Event-Driven Orchestration (Airflow Datasets) — replaced cron schedules with data-aware triggers, enabling instant downstream updates
  • Scalable OOP Architecture — refactored pipeline using dependency injection and dataclasses, enabling unit testing without cluster dependency
  • Idempotent Backfilling System — designed a safe backfill mode for historical data reprocessing without impacting production workloads
Data Quality
  • Idempotent write design guarantees no duplicates on retries or backfills
  • Partition-level validation checks confirm completeness before downstream DAGs trigger
  • Late-arriving data detection via Delta Lake CDF ensures historical partitions are recomputed accurately rather than silently stale
  • Unit-testable pipeline architecture (OOP + dataclasses) enables quality assertions to run without a live Spark cluster
Impact
  • Eliminated full table scans — significant reduction in compute costs
  • Reduced DAG execution time and improved overall pipeline efficiency
  • Delivered accurate, near real-time revenue metrics for analytics and Tableau reporting
  • Enabled reliable LTV and ROAS calculations for business decision-making
